[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the underlying architecture. When a user sends a request, it always goes through our Security Checkpoint node first, which screens for PII and prompt injections. If cleared, it flows to the orchestrator_agent. The orchestrator delegates analysis to three specialized sub-agents: the expense_tracker_agent, the budget_planner_agent, and the subscription_manager_agent. These sub-agents query financial data via our stdio MCP Server. If any sub-agent suggests a high-value action, the flow passes through a review_gate that pauses for user confirmation before returning the final_output.
